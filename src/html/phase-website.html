USER INTERFACE CASE STUDY

Phase Robotics - Web Interface

CHAPTER 1

Research &
Development

The 'PhaseRobotics' website overhaul came as part of a wider package involving an application interface, as can be seen on the home page of the site. 'Phase' is an automated purchasing solution that automatically checkouts 'hyped' items of clothing and shoes at their retail price, which users can sell on for profit. Because of this, Phase has an impossibly particular user base that presents unparalleled challenges in terms of design philosophy.

Despite knowing that I would also be redoing the user interface for the application itself, I came into this project with a fresh set of research and development strategies. Knowing that the target audience will be users of the application, layout and information hierarchy were pivotal in conveying what the application does.

It was important to use relevant terminology without alienating potential new users that might not be as aware of what the software is capable of doing. When I looked at similar sites, with a similar goal, infographics were used quite frequently to show important features of the software; making them a focal point of the design is key to users understanding the purpose of the software.

CHAPTER 2

Colour Palette

The colour palette of the website has taken a similar form to that of the dashboard-style interface. Continuity through the designs is pivotal for an interconnected design system and recognisable brand, as well as helping the user to establish an important connection between the two.

Despite a lot of the colours being the same, I have added some variance that is specific to the website. In this instance, I have used a light-ish green colour as the background for primary calls to action and also where important information is displayed, the brightness of this colour contrasts well on the dark background so allows the user's attention to be drawn towards it.

CHAPTER 3

Typeface

The type I used for the website is different from the one used in the actual software for a few different reasons. Firstly, it suits a website context better, it displays larger groups of text better due to its better legibility at smaller font sizes/weights.

Not only this but it also more welcoming to potential new users who are looking into the software for the first time and are looking for easily legible and digestible information.
